# 模組二:揮拍姿勢偵測(姿態估計版)

> **這是「追揮拍動作」版本。** 目標是球員手臂/球拍的揮拍路徑,不是球的飛行。
> 若哪天改要追**球的飛行軌跡**,這份要換回 TrackNet 版(熱圖偵測高速小球),模組一、三大致沿用。

## 概覽

職責:**逐幀回答「揮拍者的手臂(肩—肘—腕)現在在畫面哪裡」**,輸出逐幀關節座標。手腕(或由手腕外推的拍頭)隨時間的軌跡,就是揮拍動作軌跡。串接、平滑、算速度是下一個模組的事。

離線分析,不必管推論速度,可以用較準的設定多跑幾遍。

## 快速參考

| 任務 | 做法 |
|------|------|
| 抓人體關節 | 姿態估計(MediaPipe Pose),取肩—肘—腕揮拍鏈 |
| 揮拍軌跡主體 | 慣用手的**手腕**(雙手反拍取兩手腕中點) |
| 要拍頭軌跡 | 手腕往拍面方向**外推**,或另做球拍偵測(MediaPipe 不含球拍) |
| 多人畫面 | 鎖定要分析的那一位(面積最大 / 距中心最近 / 首幀指定) |
| 先驗證 | 拿預訓練模型**直跑**,肉眼看手腕軌跡品質 |

## 為什麼用姿態估計,而非 TrackNet

追的目標從「又小又快的球」換成「畫面裡夠大的人體」,難點完全不同:

- 人體關節有成熟的預訓練模型(MediaPipe Pose 在海量、多樣的人體資料上訓練過),對機位、光線、背景**遠比球偵測耐受**。
- 所以前面討論球偵測時那套「預訓練泛化不良、八成要微調」的惡夢,在這裡**大幅減輕**——多數情況預訓練直接堪用。

## 開發順序:先直跑,通常不必微調

1. **拿現成 MediaPipe Pose 直跑**,在幾支真實影片上看**手腕軌跡品質**。零成本。
2. 多半堪用就直接進下個模組。會卡的通常不是模型不準,而是**輸入條件**(人太小、被遮擋、多人選錯),回頭看模組一的合格檢查。
3. **要拍頭(而非手腕)軌跡才需要加工**:MediaPipe Pose 只給身體關節,沒有球拍。兩條路——(a) 從手腕沿前臂方向外推一段固定長度當作拍頭近似(簡單但粗略);(b) 另外訓一個球拍偵測器抓拍頭(準但要標註資料)。先用 (a) 跑通,有需要再上 (b)。

## 取哪些關節

- **核心:慣用手的肩—肘—腕**(讀 metadata 的 `dominant_hand` 決定取左或右)。手腕是軌跡主體,肩、肘用來看揮拍鏈的角度與發力順序。
- **雙手反拍要取兩側手腕**:若 `two_handed_backhand` 為真,打反拍時非慣用手也握在拍上,兩隻手腕一起動。這時可取兩手腕的**中點**當握把代表,或兩條都留;正手仍以慣用手為主。依 `stroke_type` 切換取法。
- **發球(serve)是特例**:大幅度過頭揮拍,撓背位置手臂繞到頭後會短暫自我遮擋,關節信心會掉——這很正常,別誤判成偵測失敗,交給模組三用前後幀補。
- **可選:髖、膝、踝**——想分析重心轉移與步法時加進來。

## 與球偵測不同的新難點

- **球員要夠大、夠完整入鏡**:跟球偵測「拍遠看全場」相反,這裡要人佔足夠畫面、揮拍側別長時間被遮擋(由模組一把關)。
- **多人要選對人**:畫面有對手/觀眾時,要穩定鎖定同一位,別在幀間跳人。
- **2D vs 3D**:MediaPipe 給的是 2D(或偽 3D)。揮拍平面若斜對鏡頭,軌跡會被投影壓縮失真;要真實 3D 揮拍平面需多機位或深度資訊。輸出時要講清楚這是「畫面投影軌跡」還是「真實 3D」。
- **發球的自我遮擋**:網球發球的撓背階段手臂藏到頭/背後,姿態估計會短暫失準或低信心。這是網球最常見的遮擋來源,務必保留低信心幀讓模組三補,別丟掉。
- **快速揮拍仍會模糊**:接觸瞬間手臂極快(發球尤甚),低 fps 下手腕定位會抖或漂,fps 一樣重要(由模組一把關)。

## 輸出契約(交給軌跡模組)

逐幀輸出,固定格式並**存檔**:

- 每幀:`frame_idx`、各取用關節的座標 + **信心值**、是否有效偵測。
- 信心過低 / 被遮擋的幀**保留並標記**(別跳過),下個模組要靠前後幀做修補與平滑。
- 若做了拍頭外推,要另存一條「拍頭(近似)」序列,並註明是外推值而非實測關節。
- 建議存成可重跑的檔(JSON / parquet),改下游邏輯不必重跑姿態推論。

## 雷區與提醒

- **手腕 ≠ 拍頭,別混為一談**:沒做外推/球拍偵測時,你拿到的是手腕路徑,不是拍頭路徑,輸出與用詞要誠實標明。
- **左右手 / 鏡像 / 雙手反拍**:依 `dominant_hand` 取對側關節,雙手反拍時別漏掉非慣用手;注意有些影片左右翻轉,會把慣用手取錯。
- **模型只負責「在哪裡」**:揮拍階段切分、平滑、速度都留給模組三,這層別硬塞動作判讀。
- **可重現性**:記錄模型版本與設定,固定參數,確保同影片每次結果一致。
- **別只看一支影片**:用幾支不同機位/距離的影片評估,才知道「人要多大、角度多正」才穩。
